# NetDiag

Асинхронная утилита на базе Python3 для диагностики, автоопределения, регистрации и активного управления абонентскими терминалами (ONU) на OLT-коммутаторах ZTE C300 и ZTE C600.

## Ключевые возможности

### 1. Работа с GPON сетями (ZTE C300/C600)
- **Параллельный опрос сети OLT**: Высокоскоростной одновременный сбор списка незарегистрированных (unconfigured) ONU по всему пулу OLT-коммутаторов с использованием асинхронного ввода-вывода (`asyncio`).
- **Автоопределение положения ONU (Auto-Discovery)**: Поиск OLT-коммутатора и конкретного физического PON-порта, на котором зарегистрирован или ожидает регистрации абонент, по его серийному номеру.
- **Глубокая диагностика**: Сбор уровней PON-сигналов (Tx/Rx), линков Ethernet-порта, прокачки трафика, IP/MAC-адресов абонентского оборудования и истории аварий за одну сессию.
- **Интерактивная регистрация (`--reg`)**: Поиск неконфигурированной ONU и её автоматическая регистрация на OLT C600 и C300. Поддерживается автоматический откат к совместимому профилю для разных версий прошивок, а также специфическая конфигурация для аппарата `192.168.2.13` (2.13) под нужды конкретных моделей (ZTE-F601, ручные vport и relay-агенты).
- **Удаленное управление**: 
  - Удаленная OMCI-перезагрузка ONU (`--reboot`).
  - Полная деконфигурация (удаление) ONU с интерфейса OLT (`--remove`).

### 2. Работа с коммутаторами доступа L2 (IPoE ZTE ZXR10)
- **Диагностика порта абонента**: Сбор данных о состоянии физического линка (Up/Down, скорость), уровне текущей утилизации полосы (Input/Output в %), количестве MAC-адресов на порту, привязках IP по DHCP Relay и системных логах коммутатора.
- **Счетчики ошибок**: Вывод в реальном времени количества ошибок на приеме (`InMACRcvErr`) и контрольных сумм (`CrcError`).
- **Интеллектуальная аналитика**: 
  - Детекция кабельных повреждений (CRC-ошибки).
  - Определение ситуаций, когда абонент воткнул кабель провайдера в LAN-порт своего роутера вместо WAN (обнаружение более одного MAC-адреса на порту).
  - Детекция финансовой блокировки (нахождение абонента в изоляции/заглушке по диапазону полученного IP-адреса).
  - Определение частого падения линка ("флапа порта") на основе объема записей в кольцевом буфере логов.
- **Управление портом (`--reboot`)**: Удаленный перезапуск порта (Disable -> Enable), сброс защиты MAC Protect (очистка режима Restrict при превышении лимита MAC-адресов) и обнуление счетчиков ошибок на порту.

## Системные требования
- Python 3.8+
- Зависимости отсутствуют (утилита построена исключительно на стандартной библиотеке Python).

## Быстрый старт
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/ZephronixQ/NetDiag.git
   cd NetDiag
   ```
2. Создайте файл `config.py` и заполните массив `OLT_DEVICES` параметрами ваших коммутаторов. 

## Использование

### 1. Вывод справки
```bash
python main.py --help
```

### 2. Работа с GPON OLT (ZTE C300/C600)

**Поиск незарегистрированных ONU**
Параллельный асинхронный опрос всех OLT, указанных в конфигурационном файле:
```bash
python main.py --uncfg
```

**Запуск диагностики абонента**
Скрипт автоматически найдет нужный OLT и соберет аналитику по серийному номеру:
```bash
python main.py --gpon XPON57000562
```

**Удаленная перезагрузка ONU**
```bash
python main.py --gpon XPON57000562 --reboot
```

**Удаление конфигурации ONU с порта OLT**
```bash
python main.py --gpon XPON57000562 --remove
```

**Регистрация новой ONU**
```bash
python main.py --reg XPON57001520 --vlan 1916 --int 1/3/8 --port 55 --id 10022870
```

### 3. Работа с коммутаторами доступа L2 (IPoE ZTE)

Диагностика и управление абонентскими портами на коммутаторах серий ZTE ZXR10.

**Диагностика порта абонента**
Номер порта коммутатора передается быстрым позиционным аргументом сразу после IP:
```bash
python main.py --ipoe 192.168.6.200 2
```

**Сброс ошибок, блокировки MAC Protect и перезагрузка порта**
```bash
python main.py --ipoe 192.168.6.200 2 --reboot
```
