# Roadmap развития проекта

Основные возможности мониторинга и активного конфигурирования GPON-сетей и коммутаторов доступа IPoE успешно интегрированы в CLI-интерфейс.

## Выполнено в релизах 1.0.0 - 1.3.2
- [x] **Параллельный опрос сети GPON (`--uncfg` / `-u`)**: Высокоскоростной асинхронный парсинг неконфигурированных ONU по всем OLT.
- [x] **Автоопределение ONU (`--gpon` / `-g`)**: Динамический поиск OLT по серийному номеру абонента.
- [x] **Удаленная перезагрузка ONU (`--reboot` GPON)**: Принудительный перезапуск ONU с обработкой подтверждений со стороны OLT.
- [x] **Удаление ONU (`--remove`)**: Полная деконфигурация ONU с порта OLT.
- [x] **Упрощенная Zero-Config регистрация ONU (`--reg` / `-r`)**: Автоматическое определение OLT, интерфейса, свободного индекса ONU и VLAN порта. Поддержка позиционного ввода и коротких флагов (`-i`, `-p`, `-v`).
- [x] **Диагностика IPoE-абонентов (`--ipoe`)**: Сбор метрик, MAC-адресов, DHCP-привязок и логов L2-коммутаторов доступа ZTE серий ZXR10.
- [x] **Снятие блокировок MAC Protect на коммутаторах**: Разблокировка портов в режиме Restrict и сброс статистики ошибок одной командой.
- [x] **Оптимизация архитектуры кода (Рефакторинг)**:
  - [x] Разделение парсеров на изолированные модули (`device.py`, `port.py`, `mac.py`, `dhcp.py`, `logs.py`).
  - [x] Вынос конфигурации CLI-аргументов из главного исполняемого файла в `cli.py`.
  - [x] Разделение уровней абстракции в IPoE-диагностике (выделены уровни `session.py`, `actions.py` и сценарии `flow.py`).
- [x] **Интеллектуальная база данных неконфигурированных ONU**:
  - [x] Интеграция базы данных SQLite для контроля истории и времени появления неконфигурированных ONU.
  - [x] Разделение вывода на новые (`--uncfg`) и «зависшие» устройства (`--uncfg-old`).
  - [x] Разработка самообучающегося механизма связывания серийных номеров ONU с номерами договоров абонентов при диагностике и регистрации.
