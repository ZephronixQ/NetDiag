# Changelog

Все важные изменения этого проекта будут документироваться в данном файле.

## [1.3.0] - 2026-07-08
### Добавлено
- Внедрено постоянное хранилище на базе SQLite (`diagnostic_history.db`) для отслеживания истории неконфигурированных ONU и хранения кэша абонентских договоров.
- Новый CLI-параметр `--uncfg-old` для вывода незарегистрированных ONU, находящихся в сети более 24 часов. Таблица включает информацию о дате первого обнаружения и последнем известном договоре абонента.
- Механизм автоматического «обучения» кэша договоров: при запуске общей диагностики абонента (`--gpon`) или выполнении успешной регистрации (`--reg`), скрипт сохраняет пару `Серийный номер -> Договор (User-Defined-ID)` в базу данных.
- Механизм автоматической очистки: при успешной регистрации ONU или её отключении от физического PON-порта, запись об устройстве автоматически удаляется из таблицы временного мониторинга незарегистрированных устройств.

### Изменено
- Изменен вывод команды `--uncfg`: теперь она отображает только новые ONU, находящиеся в сети менее 24 часов (без перегрузки интерфейса старыми записями).

## [1.2.1] - 2026-07-07
### Добавлено
- Новая постоянная таблица `⚠️ PORT ERRORS` в выводе диагностики IPoE (`renderer.py`), отображающая счетчики `IN-MAC-ERR` и `CRC-ERR` при каждом запуске.

### Изменено (Рефакторинг кодовой базы)
- **Декуплинг CLI**: Логика парсинга аргументов командной строки (`argparse`) и текст интерактивной справки полностью вынесены из `main.py` в файл `cli.py`.
- **Модульные парсеры IPoE**: Устаревший монолитный файл `core/operations/switch/parsers.py` разделен на независимые модули по принципу SRP (Single Responsibility Principle):
  - `parsers/base.py` — общие низкоуровневые функции очистки ANSI/IAC и регулярные выражения.
  - `parsers/device.py` — разбор информации о шасси и модели ZTE.
  - `parsers/port.py` — парсинг состояния, скорости линка и утилизации.
  - `parsers/mac.py` — разбор таблицы динамических адресов и статуса MAC Protect.
  - `parsers/dhcp.py` — обработка DHCP Relay binding.
  - `parsers/logs.py` — фильтрация системных логов по границе слова порта (`\b`).
- **Разделение транспортного и прикладного уровня**: Модуль управления коммутаторами разделен на три уровня абстракции для упрощения тестирования:
  - `session.py` — работа с сокетами, IAC-опциями Telnet и пагинацией.
  - `actions.py` — формирование и последовательное выполнение команд диагностики/перезагрузки.
  - `flow.py` — высокоуровневый сценарий запуска, координация пауз и отрисовка результатов.
- **Оптимизация `main.py`**: Файл избавлен от прямой логики протоколов связи и сокращен до ~60 строк чистого роутинга команд.

## [1.2.0] - 2026-07-07
### Добавлено
- Функционал диагностики IPoE-абонентов на коммутаторах доступа ZTE (`--ipoe <IP> <PORT_ID>`).
- Команда сброса блокировок MAC Protect, очистки счетчиков CRC/MAC-ошибок и перезагрузки порта коммутатора доступа (`--ipoe <IP> <PORT_ID> --reboot`).
- Высокоскоростной асинхронный последовательный опрос коммутаторов `exec_cmd` с моментальным перехватом приглашения `#`, выполняющий опрос за ~1 секунду.
- Аналитическая подсистема для L2-коммутаторов: детекция кабельных ошибок, схем неверного подключения домашнего роутера (кабель в LAN вместо WAN) и детекция нахождения абонента в финансовой блокировке (заглушке) по DHCP-адресу.
- Точная фильтрация логов с отсечением схожих номеров портов коммутатора на уровне Python (регулярное выражение с границей слова `\b`).

## [1.1.0] - 2026-07-06
### Добавлено
- Функционал активного управления ONU: удаленная OMCI-перезагрузка (`--reboot`) и удаление конфигурации с OLT порта (`--remove`).
- Мастер автоматической регистрации ONU (`--reg`) с поддержкой динамического определения нужного OLT по unconfigured-списку.
- Автоматическая обработка синтаксиса на OLT C300: резервный откат конфигурации при отсутствии поддержки опции `remote VLAN` на старых прошивках.
- Специфический сценарий регистрации для коммутатора `192.168.2.13` (2.13): интеграция модели ZTE-F601, кастомных vport, лимитов, dhcpv4-relay и корректного извлечения RID через `show port-identification`.
- Функция автоматического сохранения конфигурации (`write`) на OLT после завершения сессии регистрации.

### Исправлено
- Исправлен парсер неконфигурированных ONU на платформе ZTE C600, который ранее игнорировал строки, содержащие названия моделей (например, `MTE-X100DG`) между портом и серийным номером.

## [1.0.0] - 2026-07-05
### Добавлено
- Первая стабильная версия (Initial Release).
- Модульная архитектура с разделением логики под платформы ZTE C300 и ZTE C600.
- Функция параллельного опроса сети OLT для быстрого поиска незарегистрированных ONU (`--uncfg`).
- Функция автоопределения расположения ONU по серийному номеру при запуске диагностики (`--gpon`).
- Модуль автоматической аналитики показателей затухания, линков Ethernet-порта и логов аварий.
